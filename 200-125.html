OSPF Neighbor Simlet.   Based on this 9tut question.

This simlet is a set of four questions about basic troubleshooting of an OSPF network. Remember that we are not supposed to reconfigure the shown routers in order to solve the issues. We just need to demonstrate that we have figured out what the problems are. In reality, we are answering four multiple choice questions. However, we still have to type some commands on the routers, otherwise we won't get full credit for the answers, even if they are correctly answered.

Note:   Type show run on every single router. And run show ip ospf int <interface > on the "problematic" interfaces, like on Router4 S1/0, Router3 S1/0, Router3 S1/1, Router5 S1/0. Again, review the posted link above.

Question 1:    After typing the show commands, we discover that there's an area mismatch:   interface S1/0 of R3 is in area 0 while interface s1/0 of R4 is in area 2.


Question 2:    The ospf hello-interval is different in R3 and R5. It's 50 on R3 and 10 (default value) in R5.


Question 3:   After running show run on R1 and R2, we could see that the hello-interval and dead-interval are different. That's the problem.


Question 4:   The real problem here is that R6 uses the same router-id of R3 (192.168.3.3) so OSPF neighborship cannot be established.
