Access Control Lists.


Exercise 1:

Download the starting template from right here. Take note of the IP addresses and verify that all PCs can access all servers. Create ACLs to meet these requirements:

  • No computers from the 192.168.1.0/24 network should be able to access the Web Server (192.168.3.150).

  • No computers from the 192.168.2.0/24 network should be able to access the FTP service running on 192.168.3.200.

  • All other traffic that is not denied is allowed from those two networks.

  • Verify that the conditions have been met.


Exercise 2:

Download the starting template from right here. Create a record in the DNS Server (192.168.3.100) by adding a new entry: 192.168.3.200 -> www.github.xyz. Test it from the 192.168.1.10 PC and from the 192.168.2.20 PC using their web browsers or by simply pinging the host name from them.

Create a named ACL on the right router to block DNS traffic from computers 192.168.1.30 and 192.168.2.30. Verify that it works.

Exercise 3:

Modify the ACL created in the previous exercise. Deny ICMP traffic from any PC to any server. All other traffic is allowed. Verify the ACL by using ping and tracert.

Exercise 4:

Download the starting template from right here. Configure a scope on the DHCP Server for the 192.168.1.0 /24 network.

Configure the left router to forward DHCP requests to the DHCP Server by using the ip helper-address a.b.c.d command on the interface listening to those requests (gig0/0).

Verify that the DHCP clients obtained an IP address. You may need to run ipconfig /renew.

Exercise 5:

Using the .pkt file from the previous exercise, create a named ACL on any of the routers to block DHCP traffic coming from the left network. You should first release and then renew the PC IP addresses to verify that the ACL actually works.
