PowerShell Networking Commands - Part II.



    Ping with PowerShell


  1. Simple ping :  Test-Connection www.akamai.com


  2. Pinging multiple IP addresses :  28..31 | % { Test-NetConnection -ComputerName 206.13.$_.12 } | FT -AutoSize




    Tracing the route to a target.


  1. TraceRoute (similar to tracert.exe) :  Test-NetConnection www.yahoo.co.jp -TraceRoute




    Port scanning.


  1. Is port 80 open? :  Test-NetConnection -ComputerName www.idt.edu -Port 80


  2. Is port 80 open? :  Test-NetConnection -ComputerName www.idt.edu -CommonTCPPort HTTP



  3. Is port 80 open? :  Test-NetConnection -ComputerName www.gratisexam.com -Port 80


  4. Is port 80 open? :  Test-NetConnection -ComputerName www.gratisexam.com -CommonTCPPort HTTP



  5. Is port 22 open? :  Test-NetConnection -ComputerName 104.62.30.178 -Port 22




    Querying DNS (there are better tools, but PS is OK for this task).


  1. Name Server Lookup : 
    Resolve-DnsName www.wsj.com
    Resolve-DnsName www.nytimes.com
    Resolve-DnsName www.lapdonline.org



  2. Name Server Lookup for specific DNS records : 
    Resolve-DnsName www.wsj.com MX -Server 1.1.1.1 | format-list
    Resolve-DnsName www.wsj.com MX -Server 8.8.8.8 | format-list
    Resolve-DnsName www.nytimes.com MX -Server 1.1.1.1 | format-list
    Resolve-DnsName www.lapdonline.org MX -Server 1.1.1.1 | format-list
    Resolve-DnsName www.idtlearning.com -Type MX -Server 1.1.1.1 | format-list
