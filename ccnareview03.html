Working with named extended Access Control Lists




Download the starting .pkt from right here.

Step 1:

Verify that the PCs can ping the server and that they can access the FTP and HTTP services.

Step 2:

Create a named ACL that will block access from the subnets where the PCs are to the HTTP/FTP server.

Step 3:

Verify that the ACL worked.


Commands to be written on the right router:

enable

configure terminal

ip access-list extended block_http_ftp

10 deny tcp 192.168.1.0 0.0.0.255 host 192.168.3.100 eq 80

20 deny tcp 192.168.1.0 0.0.0.255 host 192.168.3.100 eq 20

30 deny tcp 192.168.1.0 0.0.0.255 host 192.168.3.100 eq 21

200 permit ip any any

exit

int gig0/1

ip access-group block_http_ftp in

end

show access-lists

copy run start
