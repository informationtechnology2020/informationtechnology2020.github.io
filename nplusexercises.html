Network+ Exercises.

These exercises have to be done on the command line interface. Optional: Change the terminal color and font size to make it more comfortable.

  1. Find your computer name by typing: hostname


  2. Find your login user account: whoami


  3. Find your IP address, subnet mask and default gateway: ipconfig


  4. Find your DHCP and DNS servers: ipconfig /all . A more specific way is shown in the next two exercises below:


  5. ipconfig /all | find /i "DHCP Server"


  6. ipconfig /all | find /i "DNS Server"


  7. Release your IPv4 address (and renew it): ipconfig /release & ipconfig /renew


  8. Release your IPv6 address (and renew it): ipconfig /release6 & ipconfig /renew6


  9. Show your DNS cached information: ipconfig /displaydns


  10. Clear your DNS cached information: ipconfig /flushdns


  11. Register your DNS server (needed when you have to use a new DNS server): ipconfig /registerdns . You may be required to "elevate" your account. Just right click on the cmd.exe and run it as an administrator.


  12. Ping your default gateway IP address (I'm assuming it's 192.168.7.254, but yours may be different): ping 192.168.7.254


  13. Ping your default gateway 65 times: ping -n 65 192.168.7.254 . You can stop the pinging by pressing CTRL C.


  14. Ping your default gateway 100 times and send/receive 1000 bytes (instead of 32): ping -n 65 -l 1000 192.168.7.254


  15. Ping the Wall Street Journal web site: ping www.wsj.com


  16. Ping the Wall Street Journal web site using IPv4: ping -4 www.wsj.com


  17. Ping the Wall Street Journal web site using IPv6: ping -6 www.wsj.com


  18. Find the host name for this IP address 1.1.1.1: ping -a 1.1.1.1


  19. Find the host name for this IP address 8.8.8.8: ping -a 8.8.8.8


  20. Find the host name for this IP address 8.8.4.4: ping -a 8.8.4.4


  21. Find the host name for this IP address 206.13.29.12: ping -a 206.13.29.12


  22. Ping https://www.intelligence.gov/ permanently: ping -t www.intelligence.gov/


  23. Using the netsh.exe command to find out your IP address(es): netsh interface ip show address


  24. Filtering the output of the netsh.exe to get your IP address(es): netsh interface ip show address | findstr "IP Address"


  25. Using the WMIC (Windows Management Instrumentation Command Line) to find your IP address(es): wmic NICCONFIG WHERE IPEnabled=true GET IPAddress . Do not worry too much about this command. Remember that we said in class, hopefully you were there paying attention, that the wmic command is being deprecated. The recommended way to work on the terminal is using the PowerShell.


  26. Get your MAC address (your NIC's physical address): getmac


  27. Get your MAC address (your NIC's physical address): ipconfig /all | findstr "Physical"


  28. Find the logical and physical IP addresses of your local subnet: arp -g


  29. Find the logical and physical IP addresses of your local subnet: arp -a


  30. Finding NetBIOS over TCP/IP statistics: nbtstat -n


  31. Finding NetBIOS over TCP/IP statistics every five seconds: nbtstat -n 5


  32. Using netstat.exe for displaying protocol statistics and current TCP/IP network connections: netstat


  33. Using netstat switches to get more information about your network connections: netstat -ano


  34. To get even more information using netstat.exe (it requires elevation): netstat -b


  35. To get even MORE information using netstat.exe (it requires elevation): netstat -bano


  36. To trace the route taken from your computer to cadem.org: tracert cadem.org


  37. To trace the route taken from your computer to cagop.org: tracert cagop.org


  38. To get information about network latency and network loss at intermediate hops between a source (your computer) and destination: pathping cve.mitre.org . Be patient, it will take some minutes.


  39. To ping a range of IP address (just once per IP), starting at 54.236.189.60 thru 54.236.189.67:     for /l %i in (60,1,67) do ping -a -n 1 54.236.189.%i


  40. Using telnet to have some fun (see picture below to know how to install telnet on your system): telnet towel.blinkenlights.nl 23
