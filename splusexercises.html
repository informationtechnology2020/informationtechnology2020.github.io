Exercise 3:  Using a keylogger (written in C#).
What's a keylogger?

  1. Go to the command line and type:
    • cd %homedrive%%homepath%\Desktop
    • mkdir Spy
    • cd Spy
    • type nul > svchost.txt
  1. Open the svchost.txt file with notepad and copy/paste the C# code from here. Save the file and close it. Close the command line interface too.
  2. Go back to the command line and type: sysdm.cpl. Go to Advanced, Environment Variables, Path, Edit, New and add this: ;c:\windows\microsoft.net\framework64\v4.0.30319; and click OK. Notice that the PATH variable is both on the user variables and the system variables. So you have to modify both. To verify that you did it right, go back to the command line and type: path. You should be able to see the entry you just typed. You may need to first close the command prompt.
  3. Go back to the command prompt and type:
    cd %homedrive%%homepath%\Desktop\Spy
    csc svchost
    dir
    If you now have two files (svchost.txt and svchost.exe) then you can continue with the exercise.
  4. Still on the command line (at the same place you were before) type:
    tasklist | find /i "svchost". Notice that there are a lot of svchost.exe running on the background.
  5. Type this on the command prompt: svchost.exe. Now the keylogger is running...
  6. Open a web browser and visit https://mail.yahoo.com/. Sign in as bradparscale and use any long password you want. It won't work. We are just testing the keylogger.
  7. Go back again to the command prompt and type:
    type kgb4u.txt
    Notice that the keylogger recorded what you've typed. Close the file and type tasklist | find /i "svchost". Do you notice the new svchost.exe (the keylogger) running?
  8. Visit https://www.virustotal.com/gui/home/upload and upload the svchost.exe file. What info did you get about the file?
  9. C'est fini.

Exercise 4:  Using the NMAP (Network Mapper) tool.
What's NMAP?

  1. Download NMAP from here and install it. Use common sense if prompted for any parallel installation of additional software.
  2. Go to the command prompt and enjoy using this great tool. Type: nmap -6 -F mail.yahoo.com
    What ports are open? Which other addresses were detected, but not scanned?
  3. Visit this website https://urlscan.io/result/2fb9ffb7-474c-4aa3-8284-24079ec6f933 and notice the IPv6 address and to whom it belongs.
  4. Now type:
    nmap -F -6 2606:4700::6812:9444
    What ports are open? Is that common?
  5. And now type (paying attention to open ports, additional IP addresses):
    nmap -F -6 www.berniesanders.com
    nmap -F -6 www.elizabethwarren.com
    nmap -F www.elizabethwarren.com
    nmap -F www.joebiden.com
  6. You'll find out interesting info now, just type:
    nmap -sV -T4 -O -F --version-light 192.0.66.2
    Notice the web server being used. Which OS (operating system) was most probably found? What's the probability of having found the right OS? Do you know which web site you're analyzing? FYI, just see this. Interesting?
  7. More fun (but be patient, it may take a couple of minutes):
    nmap -A 143.228.126.60
    Who's that?
    And visit https://www.ultratools.com/tools/ipWhoisLookupResult, enter 143.228.126.60 and press the Go>> orange button.
  8. Visit this website. Who's that? Just look at the very first record.
    Now a quick scan:
    nmap -T4 -F 151.101.1.127
